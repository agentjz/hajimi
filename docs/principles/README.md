# 设计原则

这一组文档不是实现细节，而是架构原则。

每篇只回答三件事：

- 这条原则是什么
- 为什么要这样设计
- 这个项目里对应到哪里

## 怎么看

如果你第一次接触这个项目，建议按 `S01 -> S12` 顺序看。

如果你已经知道这套系统大概在做什么，想直接维护代码，优先看 `../maintainers/README.md`。
