# S03 先计划再动手

## 原则
复杂任务不要直接开做，先列步骤再动手。

## 目的
避免走偏和反复返工，提高完成率与可控性。

## 在本项目中的落地
- 通过 `todo_write` 写出简短计划，且只能有一个 `in_progress`。
- 对写/改/执行类工具做硬性 gating：无计划时会被阻断并提示先 `todo_write`。
- 只读命令不受限制。
- 子智能体不强制（保持轻量、上下文干净）。

## 触发的工具
`write_file` / `edit_file` / `apply_patch` / `write_docx` / `edit_docx` / `run_shell` / `background_run`
